1、Android程序结构

1、app:用于存放程序的代码和资源内容

  • manifests
    用于存放AndroidManifest xml文件(又称清单文件).该文件是整个项目的配置文件。在程序中定义的四大组件都需要在这个文件中注册，另外在该文件中还可以给程序添加权限。在清单文件中配置的信息会添加到Android 系统中，当程序运行时，系统会找到清单文件中的配置信息，然后根据配置信息打开相应组件。

  • java
    用于存放所有的Java代码，在该文件夹中可以创建多个包，每个包中可以存放不同的文件或Activity。

  • res
    用于存 放Android 程序所用到的资源， 例如图片布局文件、字符串等。
    drawable 目录用于存放图片及XML文件，
    layout目录用于存放布局文件，
    mipmap目录通常用于存放应用程序图标，系统会根据手机屏幕分辨率(hdpi/mdpi/xhdpi/xxhdpi/xxxhdpi)匹配相应大小的图标，
    values目录用于放置定义的字符串。

  • Gradle Scripts
    用于存放项目创建的相关文件， 无须修改

2、资源的管理和使用

res——资源文件

调用资源文件的两种方式：

1、java代码（在Activity中通过gerResources().getDrawable()方法调用图片资源）

getResources().getDrawable(R.mipmap.ic_launcher); //调用mipmap文件夹中的资源

getResources().getDrawable(R.drawable.ic_launcher); //调用drawable开头的文件夹中的资源

2、在xml布局中调用图片资源

@mipmap/ic_launcher

@drawable/icon

主题和样式资源

1、主题（res/values/style.xml）

<resources>

    <!-- Base application theme. -->
    <style name="AppTheme" parent="Theme.AppCompat.Light.DarkActionBar">
        <!-- Customize your theme here. -->
        <item name="colorPrimary">@color/colorPrimary</item>
        <item name="colorPrimaryDark">@color/colorPrimaryDark</item>
        <item name="colorAccent">@color/colorAccent</item>
    </style>

</resources>

< resources >中可以有多个 < style >,可以在AndroidMainifest.xml中通过name=”Apprheme”,去设置主题

<application ...android:theme = "@style/AppTheme"> ...</application>

在java代码中设置主题

setTheme(R.style.AppTheme)

2.样式（res/values/style.xml）

<style name="textViewSytle">
    <item name="android:layout_weight">20dp</item>
    <item name="android:layout_height">20dp</item>
    <item name="android:background">#f54e39</item>
</style>

在布局文件的View控件中调用

<TextView...style="@style/textViewSytle"/>

3、布局文件（res/layout/activity_main.xml）

用于搭建各个界面（关于样子的代码都在这里调用）

4、字符串资源（res/valuse/string.xml）

<resources>
    <string name="app_name">helloworld</string>
</resources>

5、颜色资源（res/valuse/colors.xml）

<resources>
    <color name="colorPrimary">#008577</color>
    <color name="colorPrimaryDark">#00574B</color>
    <color name="colorAccent">#D81B60</color>
</resources>

6、尺寸资源（res/valuse/dimens.xml）在android studio3.2中需要手动创建

注意：android支持的尺寸单位
px:像素
dp：设备独立像素，，和px一样，不过可以在不同大小的机器上按比例缩放
sp：比例像素，主要处理文字大小
in：英寸
pt：磅
mm：毫米

三、Android常见界面布局

1、View视图

每个界面的根元素必须有且仅有一个ViewGroup

2、界面布局的编写方式

在xml中编写布局

布局文件通常放在acticity_main.xml中

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintLeft_toLeftOf="parent"
        app:layout_constraintRight_toRightOf="parent"
        app:layout_constraintTop_toTopOf="parent" />

</LinearLayout>

上诉代码，定义了一个线性布局LinearLayout ，在该布局中定义了一个TextView控件。其中LinearLayout 继承于ViewGroup，TextView继承自View。

在java中编写布局

并不常用，具体是通过new的方式去布局

3、常见的布局界面

  1. RelativeLayout（相对布局）
  2. LinearLayout（线性布局）
  3. FrameLayout（帧布局）
  4. TableLayout（表格布局）
  5. ConstraintLayout（约束布局）

布局的通用属性：

android:id :当前布局的唯一标识
android:layout_width :布局的宽度   （1）match-parent:宽度与父容器一致 ...
android:layout_height :布局的高度   (1) wrap_content:该布局的高度恰好能包裹它的内容
android:background="#00000000" :背景颜色
android:layout_margin :设置当前布局与屏幕边界、周围布局或空间的距离，各种有各种的属性
android:padding :设置当前布局内控件与该布局的距离
必须要有长度和宽度，其他属性可以根据需求设置

2、AlertDialog对话框的使用

使用AlertDialog创建的对话框一般包含标题，内容和按钮三个区域

普通对话框

/**
 * 重写onBackPressed()来监听回退键
 */
public void onBackPressed(){
    //申明对象
    AlertDialog dialog;
    AlertDialog.Builder builder = new AlertDialog.Builder(this)
            .setTitle("普通对话框")
            .setIcon(R.mipmap.ic_launcher)
            .setMessage("是否确定退出应用：")
            //添加"确定"按钮
            .setPositiveButton("确定", new DialogInterface.OnClickListener() {
                @Override
                public void onClick(DialogInterface dialog, int which) {
                    dialog.dismiss();  //关闭对话框
                    MainActivity.this.finish();  //关闭MainActivity
                }
            })
            //添加"取消"按钮
            .setNegativeButton("取消", new DialogInterface.OnClickListener() {
                @Override
                public void onClick(DialogInterface dialog, int which) {
                    dialog.dismiss();
                }
            });
    dialog = builder.create();
    dialog.show();

}
